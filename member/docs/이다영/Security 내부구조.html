  • 시큐리티 내부구조의 필터들은 체인으로 연결되어 순차적으로 실행되며, 각각 특정한 보안 기능을 담당합니다.
  • 필터의 순서가 중요한데, 예를 들어 인증(Authentication)은 인가(Authorization)보다 먼저 수행되어야 합니다.

내부구조

이름설명
1. 시리즈목표
2. 시큐리티 동작 원리
3. DelegatingFilterProxy/FilterChainProxy서블릿 필터와 스프링의 빈으로 등록된 필터들을 연결해주는 프록시 필터 / 보안 필터들의 목록을 관리하고 실행을 위임하는 필터
4. SecurityFilterChain 등록실제 보안 처리를 위한 필터들의 체인을 구성하고 순서를 보장
5. SecurityFilterChain 구조
6. SecurityContextHolder보안 주체(인증 정보)를 저장하고 관리하는 필터
7 : 필터 상속과 요청 전파
8. GenericFilterBean, OncePerRequestFilter요청당 한 번만 실행되는 필터 기본 구현체
9. DisableEncodeUrlFilterURL 인코딩을 비활성화하는 필터(별 기능은 없지만 세션ID가 파라미터로 유출되는 것을 막는 필터)
10 : WebAsyncManagerIntegrationFilter비동기 보안 컨텍스트를 처리하는 필터
11. SecurityContextHolderFilterSecurityContext를 요청에 바인딩하고 관리하는 필터
12 : HeaderWriterFilter보안 관련 HTTP 헤더를 추가하는 필터
13 : CorsFilterCross-Origin Resource Sharing을 처리하는 필터(GenericFilter 기반으로 구현)
14 : CsrfFilterCross-Site Request Forgery 공격을 방지하는 필터(CSRF 공격 방어를 위해, HTTP Referer 헤더를 통해 요청의 출발점, 이전 URL등을 검증)
15 : LogoutFilter로그아웃 처리를 담당하는 필터
16. UsernamePasswordAuthenticationFilter사용자명과 비밀번호 기반의 인증을 처리하는 필터
17. DefaultLoginPageGeneratingFilter기본 로그인 페이지를 생성하는 필터
18 : DefaultLogoutPageGeneratingFilter기본 로그아웃 페이지를 생성하는 필터
19 : BasicAuthenticationFilterHTTP Basic 인증을 처리하는 필터
20 : RequestCacheAwareFilter인증 전에 요청된 URL을 캐시하고 관리하는 필터
21 : SecurityContextHolderAwareRequestFilter서블릿 API 보안 메서드를 구현하는 필터
22. AnonymousAuthenticationFilter인증되지 않은 사용자를 익명 사용자로 처리하는 필터
23. ExceptionTranslationFilter스프링 시큐리티 예외를 HTTP 응답으로 변환하는 필터
